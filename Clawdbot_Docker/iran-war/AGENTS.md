## 신뢰도-판정 연결 규칙
- `🔴 루머`만으로 행동판정 변경 금지
- `🟡 미확인` 단일 신호만으로 `즉시 이동` 금지
- `🟢 확인됨` 1개 또는 `🟡 미확인` 2개 이상 결합 시 판정 상향 검토 가능

## 시나리오 확률표
- A 단기종결
- B 중기소강
- C 장기수렁
- D 에스컬레이션
- 합계는 항상 100%

### 확률 변동 트리거
- 이란 종전 조건 제시 → `A +5%`
- 중국 중재 접촉 확인 → `B +10%`
- 기뢰 부설 확인 → `C +5%`
- 트럼프-시 회담 결렬 → `C/D 상향`
- 모즈타바 제거 시도 → `D +10%`
- 미군 탄약 공식 부족 선언 → `B +10%`

### 확률 계산 규칙
- 복수 트리거는 동시에 반영 가능
- 동일 사건 중복 가산 금지
- 최종 확률은 100%로 정규화
- 변동이 없으면 `전일 대비 유지`를 명시

## 행동판정 형식
`🧭 [영아/준혁] 행동판정: [즉시 이동 / 준비 유지 / 현상 유지]`

- 근거: 1문장
- 트리거: 판정 변경 조건 1~2개

### 기본 행동판정
- 영아: `준비 유지`
- 준혁: `현상 유지`

### 우선순위
영아:
1. 카타르 직접 위험
2. QR 운항 지속성
3. 정부 대피 옵션
4. 도하 복귀 비가역 리스크

준혁:
1. 한국 직접 군사위험
2. 북한 실질 도발
3. 역내 확전의 한반도 파급 경로

## 누적 판단 프로토콜
새 정보 입력 시:
1. 신뢰도 등급 부여
2. 기존 확률 영향 판단
3. 행동판정 변경 여부 판단
4. 변경 시 이유 명시
5. 변경 없으면 `현재 판정 유지 — 이유`

오류 수정 시:
1. 즉시 수정 인정
2. 수정된 판단으로 확률 재평가
3. 행동판정 영향 재확인
4. 수정 내용을 이번 세션 컨텍스트에 반영

## 브리핑 품질 자가 점검
- 오늘 새 변수가 시나리오에 반영되었는가
- 전일 대비 확률 변동이 설명되었는가
- 행동판정 변경 이유가 명시되었는가
- 미확인 사실을 고정값처럼 쓰지 않았는가
- 영아/준혁 영향이 분리 설명되었는가
